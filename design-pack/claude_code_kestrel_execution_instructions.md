# Claude Code Execution Instructions, Kestrel Brand Rebuild and Marketing Pack

## Objective

Use the assets and guidance in the **Kestrel brand rebuild marketing pack** as the controlled source of truth and produce production-ready collateral for internal use.

This is an execution brief, not an open creative brief.

The job is to apply the approved Kestrel identity consistently across channels, preserve brand discipline, and avoid drift.

---

## Primary goal

Build and maintain a reusable Kestrel asset system across:

- email mastheads
- PowerPoint cover and section pages
- social and campaign tiles
- Teams and SharePoint banners
- PDF covers
- icon tiles and small-format digital assets

The output needs to feel:

- premium
- strategic
- modern
- signal-led
- partner-ready

It must **not** feel:

- militarised
- noisy
- over-designed
- startup-casual
- generic corporate filler

## Brand non-negotiables

### 1. Logo discipline
Use the uploaded Kestrel lock-up as the master brand.

Do not:
- redraw the bird loosely
- alter the envelope / chevron base
- re-stack the logo without explicit instruction
- substitute random typefaces for the existing wordmark
- recolour the core logo outside the approved palette

### 2. Descriptor discipline
The master descriptor remains:

**DEFENCE INSIGHTS**

Do not replace or rewrite the descriptor inside the core logo lock-up.

If a different sub-line is required for a campaign, place it **outside** the logo lock-up.

### 3. Colour discipline
Use only the approved Kestrel palette inferred from the approved logo.

Approved working palette:
- Background mist: `#EDECEF`
- Navy deep: `#2F364A`
- Navy mid: `#3A4152`
- Indigo: `#6451B3`
- Violet: `#8A6BCD`
- Royal blue: `#2568D0`
- Electric blue: `#2348AC`
- Ice: `#CDD0DF`
- White: `#FFFFFF`

### 4. Graphic language
Support graphics should come from the logic of the envelope / chevron geometry.
Use:
- angular facets
- folded planes
- restrained diagonal motion
- subtle triangular or chevron framing devices

Do not use:
- camouflage
- flags
- crosshairs
- target icons
- shield clichés unless already present in the supplied identity
- fighter jet silhouettes
- militaristic stock motifs

1. **Brand fidelity**

## Layout system

### Spacing
Keep generous whitespace.
Do not crowd the logo, headlines or callouts.

Minimum clear space around the full logo should be at least the visual height of the **K** in the wordmark.

### Alignment
Use left alignment by default.
Kestrel should feel deliberate, not decorative.

### Typography guidance
Use clean sans serif typography with a premium but restrained feel.

Safe defaults if no brand font is supplied:
- bold sans serif for headlines
- regular sans serif for body and support copy
- uppercase only where it adds control, not everywhere

### Visual density
Aim for low to medium density.
Each layout should feel easy to scan.

## Messaging rules

Kestrel means:
- sharp observation
- signal over noise
- defence market intelligence
- future-ready perspective

### Approved sample campaign line
Use this as a working sample line, not as fixed permanent brand architecture:

**Minimising future grad/hiring challenge, low-cost brand play that builds visibility and future talent pipeline**

Use Australian English.
Use direct statements.
Keep copy commercially useful.
Avoid hype.
Avoid generic consulting filler.

## Quality bar

Every asset produced should pass these checks:

### Brand fidelity
- does it clearly belong to Kestrel?
- does it preserve the uploaded approved logo correctly?

### Channel fit
- would this work in Outlook, PowerPoint, Teams or PDF without manual rescue work?

### Visual discipline
- is the spacing calm?
- are the accents controlled?
- is the logo treated with respect?

### Senior audience readiness
- would this look credible in front of a partner or director?

If the answer to any of these is no, revise it.

## Execution workflow for Claude Code

Follow this sequence.

### Step 1. Load and inspect the pack
- locate the approved logo files
- read the rebuild guide and usage rules
- confirm the palette and default asset set

### Step 2. Set the operating constraints
- brand fidelity first
- no uncontrolled redesign
- preserve logo anatomy and descriptor

### Step 3. Build or extend the required asset set
Depending on the ask, create only what is needed, using the existing pack structure.

### Step 4. Produce light and dark variants where relevant
Especially for:
- email headers
- banners
- icon tiles
- title graphics

### Step 5. Generate a preview sheet
Always create a simple preview sheet for review if more than three assets are produced.

### Step 6. Package outputs cleanly
Save all deliverables in a logically named folder and zip the result.
